# package-lock.json

---

# What is package-lock.json?

package-lock.json is automatically generated by npm.

It records the exact versions of every installed package and its dependencies.

---

# Why is it Needed?

Suppose

Developer A installs

```
Express 5.1.0
```

One month later

Developer B runs

```
npm install
```

Without package-lock.json

Developer B may receive a newer package version.

This can introduce bugs.

package-lock.json ensures everyone installs identical versions.

---

# Created Automatically

Whenever

```bash
npm install
```

is executed,

npm automatically creates package-lock.json.

---

# What Does It Store?

- Exact package versions
- Dependency tree
- Integrity hashes
- Download URLs

---

# Example

```json
{
  "name": "worksphere",
  "lockfileVersion": 3,
  "packages": {}
}
```

---

# package.json vs package-lock.json

| package.json | package-lock.json |
|--------------|------------------|
| Project metadata | Exact dependency versions |
| Editable | Auto-generated |
| Stores required packages | Stores installed versions |
| Written manually | Created by npm |

---

# Should We Commit It?

YES

Always commit package-lock.json.

Reason

Every developer should install identical dependency versions.

---

# Should We Edit It?

No.

Never manually edit package-lock.json.

# Interview Questions

## Why package-lock.json?

To ensure every developer installs the exact same dependency versions.

---

## Should package-lock.json be committed?

Yes.

Always.

---

## Can we delete package-lock.json?

Yes, but npm will recreate it.

Deleting it unnecessarily is not recommended.

---

# Best Practices

✔ Commit package-lock.json

✔ Never edit manually
